  星期六, 九月 28日 2019, 10:03 晚上

  761 字     2 分钟      次  

  这个九月真的是遇到太多太多的事情，总觉得需要去写点东西来记录一下…况且不去谈论感情或者未来发展的某某，只是对于IT行业总是有那么多新的看法和悸动，值得自己认真去反思与推敲。

  先谈一个最大的感观吧，记得上周去参加一次活动，是关于图像、视频分析的比赛，重新对于深度学习有了一个全新的了解和认识，并且结交了一位很不错的朋友，彼此交谈，不禁感叹深度学习在当今时代的重要性，同时也在思考前端对于IT行业的位置。之前和朋友去沟通，发现很多单位对于前端都属于外包类型，而很少听到会有一个单独的前端团队（BAT公司除外），所以从当前的层面来说前端的市场份额还是有些偏低的，而像很多企业对于深度学习的领域会考虑大量的人员配置，由此感慨深度学习的重要位置（纯属个人观点，若有不对之处，还望各位指正）。不过之前认识的那位朋友最近也会是给了我很多的帮助，提供了很多的学习资料并及时的做了问题的答疑，在此还是很感谢他的。
  再说一个新的事情吧，就是在这个炎热的丝毫没有太多秋意的九月，我开始了我的面试工作，值得庆幸的是我最后还是找到了一家单位。不过在段面试还是让我有了很多的反思的，所以想总结一下。

面试汇总

  框架这个话题是肯定要涉及的，目前单位主要涉及到的 Vue 、React 以及微信小程序，由于我没有做过 React 的实际项目，所以很多单位是不考虑的（这让Angular怎么想）。针对于 Vue 还是有几个点值得分享的：

路由守卫

 这个问题其实是之前没有涉及到的，就是在做路由跳转时需要做一些验证，官网地址自取：守卫导航。

后台数据请求逻辑

 一位面试官非要让我讲说 axios 的请求方式，我使用了封装的方法，结果到最后给她看了我的 demo （尴尬又不失礼貌的微笑）~

关于封装独立的 npm

 这应该算是一个很有吸引力的问题，毕竟之前都是用别人封装好的依赖包，而自己从来没有去封装过，不过这个对于技术的要求应该还是蛮高的吧？毕竟要考虑太多的因素和配置在里面，很有可能自己的配置会不全面，而影响到后期的使用。

  突然想不起来了，等想起来再更，掰掰~
